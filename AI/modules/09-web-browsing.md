# Module 09: Web Browsing (Agent Mode)

## 1. Purpose

Provide web search and page content fetching capabilities for the AI Agent mode within the insight pipeline. Uses the Serper API for Google search and custom page fetching logic for content extraction.

## 2. Actors

| Actor | Role |
|-------|------|
| Insight Pipeline | Triggers web search when `web_search` tool is selected during intent recognition |
| Serper API | Google search API for query execution |
| WebParserClient | External URL parsing service (optional) |
| Target Websites | Web pages fetched for content extraction |

## 3. Functional Requirements

| ID | Priority | Description | Verification |
|----|----------|-------------|-------------|
| FR-WB-001 | MUST | The system MUST generate web search queries from the user's insight question via LLM | T |
| FR-WB-002 | MUST | The system MUST execute web searches via the Serper API (`SERPER_API_KEY`) | T |
| FR-WB-003 | MUST | The system MUST fetch and parse web page content from search result URLs | T |
| FR-WB-004 | MUST | The system MUST handle common error indicators in fetched content (JavaScript-required pages, rate limits, auth errors) by filtering them out | T |
| FR-WB-005 | SHOULD | The system SHOULD support an optional `WebParserClient` for enhanced URL parsing via an external service | I |
| FR-WB-006 | SHOULD | The system SHOULD use concurrent fetching (ThreadPoolExecutor) for parallel page retrieval | I |
| FR-WB-007 | SHOULD | The system SHOULD handle PDF content extraction via `pdfplumber` | I |
| FR-WB-008 | MUST | The system MUST include web search results in the `external_webs` field of the insight message response | T |

## 4. Data Contract

### 4.1 Internal Interface

The web browsing module is invoked internally by the insight pipeline. No dedicated API endpoint exists.

**Input**: List of search queries (generated by LLM from user question)

**Output**: Structured search results with:
- URL
- Title
- Snippet
- Extracted page content

### 4.2 Serper API Call

```
POST https://google.serper.dev/search
Headers: X-API-KEY: {SERPER_API_KEY}
Body: {"q": "search query"}
```

### 4.3 Error Indicators (Filtered)

The module maintains a list of error indicators that signal unreliable page content:
- `'limit exceeded'`
- `'Error fetching'`
- `'Please turn on Javascript'`
- `'Enable JavaScript'`
- `'Please enable cookies'`
- and others

## 5. Processing Flow

```
Insight Pipeline (intent selects web_search tool)
  |
  v
Generate search queries (LLM call)
  |
  v
For each query:
  Serper API search -> search results
  |
  v
For each search result URL (ThreadPoolExecutor):
  Fetch page content
    |
    +-- HTML pages: BeautifulSoup parsing
    +-- PDF files: pdfplumber extraction
    +-- WebParserClient: external service (optional)
  |
  v
Filter error indicators from content
  |
  v
Package results as web evidence for LLM context
  |
  v
Include in insight response generation prompt
```

## 6. Configuration

| Env Variable | Default | Description |
|-------------|---------|-------------|
| `SERPER_API_KEY` | (required for web search) | Google Serper API key |

### Web Client Configuration

- **User-Agent**: Chrome/58.0 compatible string
- **Referer**: `https://www.google.com/`
- **Session**: `requests.Session` with persistent headers
- **WebParserClient**: Optional external URL parsing service (default URL: `http://localhost:8000`)

## 7. Error Handling

| Error Condition | Behavior |
|----------------|----------|
| Serper API key missing | Web search skipped |
| Serper API call fails | Error logged; search results empty |
| Page fetch timeout | `Timeout` exception caught; URL skipped |
| Page contains error indicators | Content filtered out as unreliable |
| PDF extraction fails | Error logged; page content empty |

## 8. Performance Considerations

- **Concurrent fetching**: Page content is fetched in parallel via ThreadPoolExecutor
- **Request timeout**: 120-second timeout for WebParserClient page parsing
- **Session reuse**: `requests.Session` reuses TCP connections for multiple page fetches
- **Content extraction**: BeautifulSoup and NLTK sentence tokenization for clean text extraction

## 9. Dependencies

| Dependency | Direction | Description |
|-----------|-----------|-------------|
| Insight pipeline | Upstream | Invoked when web_search tool is selected |
| Serper API | External | Google search provider |
| `requests` + `BeautifulSoup` | Library | HTTP fetching + HTML parsing |
| `pdfplumber` | Library | PDF content extraction |
| `nltk.tokenize` | Library | Sentence tokenization |

## 10. Open Issues

| Issue | Status | Notes |
|-------|--------|-------|
| No rate limiting on Serper API calls | Risk | High-volume insight requests could exhaust API quota |
| WebParserClient hardcoded to localhost | Minor | Default URL assumes co-located service |
| No content caching | Enhancement | Same URLs may be fetched multiple times across different insight sessions |
